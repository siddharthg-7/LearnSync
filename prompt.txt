Build a complete **Student Portal** for LearnSync using **React (functional components with hooks)** and **Tailwind CSS**.

The portal must function as a **dynamic, AI-powered learning system** that adapts based on:

* Age group
* Survey inputs
* Quiz performance
* Continuous learning behavior

All data must be managed using **localStorage (mock data)**.

The UI must follow a **clean, professional SaaS design system**:

* No gradients
* Minimal and structured layout
* Consistent spacing and typography
* Age-adaptive interface behavior

---

# CORE SYSTEM FLOW (MANDATORY)

Survey → Age Mode → Self-Assessment → 6-Question Quiz → Level Detection → Mentor Allocation → Dashboard → Continuous Learning Loop

---

# 1. DYNAMIC SURVEY SYSTEM (ENTRY POINT)

## Step 1: Basic Information

Collect:

* Name
* Age
* Class

---

## Step 2: Age Mode Detection

Define three modes:

* Age 5–10 → Foundation Mode
* Age 11–15 → Growth Mode
* Age 16–20 → Mastery Mode

This mode will control UI behavior, interaction style, and feature availability.

---

## Step 3: Subject Selection (Dynamic)

Subjects must be dynamically displayed based on the selected class:

* Lower classes → Math, English, EVS
* Middle classes → Math, Science, English, Social Studies
* Higher classes → Physics, Chemistry, Biology, Mathematics

Use a card-based selection interface.

---

## Step 4: Availability Input

Collect:

* Available days (Mon–Sun)
* Preferred time slots

This data will be used for scheduling and study planning.

---

## Step 5: Self-Assessment (Dynamic per Subject)

For each selected subject, display key topics.

Each topic must allow selection of:

* Weak
* Okay
* Strong

Example:

Fractions
[ Weak ] [ Okay ] [ Strong ]

---

## Voice Input (Optional)

Provide an option to input assessment via voice.

Example input:
"I am weak in fractions"

System should map keywords to topic-level strength.

---

# 2. 6-QUESTION QUIZ (LEVEL DETECTION)

## Purpose

To validate the student’s actual skill level based on performance.

---

## Structure

* Total Questions: 6
* Q1–Q2: Easy
* Q3–Q4: Medium
* Q5–Q6: Hard

Questions must be generated based on:

* Selected subjects
* Weak topics from self-assessment

---

## Evaluation Logic

* Score ≤ 2 → Beginner
* Score ≤ 4 → Intermediate
* Score ≥ 5 → Advanced

---

## Output

Store:

* Level
* Weak topics
* Subject preferences

---

# 3. MENTOR ALLOCATION (AI-BASED)

Assign mentors based on:

* Student level
* Subject selection
* Age group

Rules:

* Beginner students → More experienced mentors
* Younger students → Supportive and patient mentors
* Advanced students → Subject-specialized mentors

---

# 4. AGE-ADAPTIVE UI SYSTEM

## Foundation Mode (Age 5–10)

* Gamified interface
* Use stars instead of percentages
* Replace tasks with “missions”
* Large buttons and simplified navigation

---

## Growth Mode (Age 11–15)

* Balanced interface
* Include XP, badges, and progress bars
* Structured layout with mild gamification

---

## Mastery Mode (Age 16–20)

* Minimal interface
* Focus on planning, analytics, and efficiency
* No gamification elements

---

# 5. DASHBOARD (PRIMARY INTERFACE)

## Section 1: Today’s Plan

Display:

* Topics to study
* Practice tasks
* Quiz tasks

This section must be the primary focus of the dashboard.

---

## Section 2: Pending Tasks

Display remaining tasks for the day.

---

## Section 3: Courses

Display:

* Course cards
* Progress indicators

---

## Section 4: Weak Areas

Highlight weak topics.

Clicking a topic must navigate directly to the relevant topic page.

---

# 6. COURSE SYSTEM

## Structure

Course → Chapter → Topic

---

## Content Sources

* Mentor-created modules
* NGO-provided modules

---

# 7. TOPIC PAGE (LEARNING INTERFACE)

Each topic must include:

1. Explanation (AI-processed from mentor content)
2. Key Points
3. Examples
4. Summary

---

## Quiz Section

* Generated based on level and weak topics

---

## Post-Quiz Behavior

* Display score
* Provide feedback
* Update weak topics
* Update progress

---

# 8. AI CHATBOT WITH CONTEXT FORWARDING

## Feature

Allow student to send topic content to chatbot.

---

## Flow

* Topic content is passed as context
* Student asks questions based on that content

Example:
"Explain step 2 in simpler terms"

---

# 9. DOUBT SYSTEM

## Types

* Course-related doubts
* General doubts

---

## Behavior

* Doubts are broadcast to all mentors
* Filtered by subject and level

---

## UI

* List of raised doubts
* Status: Pending / Answered

---

# 10. STUDY PLANNER (FOR AGE 11+)

## Input

Student provides:

* Date
* Time
* Topics or lessons

---

## AI Processing

* Prioritize weak topics
* Distribute tasks across available time

---

## Output

* Daily plan
* Weekly schedule

---

## Integration

Study planner directly feeds into dashboard tasks.

---

# 11. MENTOR SUPPORT CONTENT

Display:

* Mentor-created modules
* Mentor-generated quizzes

---

# 12. PROFILE AND PROGRESS TRACKING

Display:

* Improvement over time
* Topic mastery progression
* Quiz scores

---

# 13. CONTINUOUS LEARNING LOOP

Learn → Quiz → Feedback → Update Profile → Generate New Plan
